## What is Version Control?

Version control lets you make changes to your agent without affecting live calls. You edit in a draft, create versions when you're ready, and publish so your agent handles calls.

## The three states of an agent

<CardGroup cols={1}>
  <Card title="Draft" icon="pencil">
    Your workspace. Saves automatically while you work, even with errors. Never handles calls.
  </Card>
  <Card title="Saved version" icon="bookmark">
    A backup copy of your agent. Doesn't handle calls, but you can publish it at any time.
  </Card>
  <Card title="Published version" icon="circle-check">
    The active version handling live calls. There can only be one per agent.
  </Card>
</CardGroup>

Each version receives an automatic incremental number (v1, v2, v3...) that never repeats.

<Note>
Saved and published versions are **immutable**. If you want to edit them, a modal will open to create a new draft from that version. The previous draft will be overwritten.
</Note>

## Workflow

<Steps>
  <Step title="Edit your agent">
    All changes you make (prompt, voice, tools, etc.) are automatically saved to the **draft**. You can modify it as many times as you want without affecting live calls.
  </Step>

  <Step title="Resolve errors">
    If your agent has configuration issues, an indicator will appear showing the **number of errors** pending. Clicking it opens a dropdown with issues organized by type:
    - **Unconfigured tools**: nodes that need a valid integration or flow.
    - **Unconnected flows**: flows that require a valid connection.
    - **Pending instructions**: conversational nodes that need instructions.
    - **Incomplete nodes**: end, transfer, or other nodes that require configuration.

    This indicator only appears when errors exist. You can keep working with errors, but **you won't be able to create a version until all of them are resolved**.

    <Frame>
      <img src="/images/errores.png" alt="Error dropdown" />
    </Frame>
  </Step>

  <Step title="Create a version">
    When there are no errors, click **Create version**. Optionally, add a descriptive name. The version is saved in the history and you can publish it now or later.
  </Step>

  <Step title="Publish">
    When you publish, the version becomes active and all new calls use it immediately. Calls in progress continue with the previous version.

    You can also publish any saved version from the history.
  </Step>
</Steps>

<Warning>
To publish a version, you must have a phone number available in your project.
</Warning>

## Version history

Within the editor you can see the complete version history to select, view, or edit any of them.

## Call tracking by version

Each call is associated with the version that handled it, allowing you to audit and compare performance across versions.
