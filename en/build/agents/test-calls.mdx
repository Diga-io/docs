## Introduction
Before putting an agent into production, it's common to want to verify it works correctly or experiment with new changes without needing to create and publish a new version for every iteration.

For this, Diga's agent editor includes an integrated testing feature that lets you simulate calls. This way, you can **interact with the agent you're editing in real time, without leaving your browser or publishing changes**.

<Frame>
  <img src="/images/agent-test.png" alt="Agent modal showing the test section to test your agent" />
</Frame>

## What is it for?

Testing your agent from the editor allows you to:

- Validate changes in the conversational flow.
- Check the agent's responses.
- Verify the behavior of tools and external integrations.

This feature is especially useful during the development phase, as it allows you to **detect errors and improve the call experience** without needing to publish a new version of the agent for every change.

<Note>
You don't need to buy or link any phone number to run tests.
</Note>

## Usage and metrics considerations

- Test calls **do not generate logs or metrics** in the system, as they are intended solely for development and validation.
- Minutes consumed during tests **do count** towards your plan's minute usage, just like real calls.

## How to test your agent

<Steps>
  <Step title="Open the test tab">
    In the side menu of the agent editor, click on the **Test** tab.
  </Step>
  <Step title="Start the test call">
    Click the **play** button. After a short load, a test call will start with the agent's current configuration.
  </Step>
  <Step title="End the test">
    When you're done, click **End test**, located at the bottom of the transcript.
  </Step>
</Steps>

There's no need to create or publish any version: you'll be interacting directly with the agent's current configuration.

<Warning>
Before starting the test, make sure your microphone and speakers are correctly configured in your browser.
</Warning>

## Differences with a real call

Test calls are not real phone calls, so some features behave differently in the testing environment.

<AccordionGroup>
  <Accordion title="Hanging up a call">
    - **Real call:** the call ends when the user hangs up.
    - **Test environment:** clicking **End test** ends the call, but you can continue viewing the transcript until you close the test tab or start a new call.
  </Accordion>
  <Accordion title="Call transfer">
    **Real call:**
    - If someone answers, the call continues with the new speaker.
    - If no one answers, the call returns to the agent.

    **Test environment:** when the agent reaches a transfer node, the call always continues with the agent, as if the transfer was not answered. This allows you to keep observing the conversational behavior during the test.
  </Accordion>
</AccordionGroup>
