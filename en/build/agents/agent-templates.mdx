## What are agent templates?

Templates are preconfigured agents designed for common use cases. Each template includes a **prompt**, an **associated toolkit** (flows, integrations, knowledge bases), and a structure ready for you to customize with your own data.

Instead of building everything from scratch, you can select a template, review its configuration, and adapt it to your business in minutes.

## How to use a template

<Steps>
    <Step title="Access the template selector">
        1. Navigate to "Agent" in the left sidebar.
        2. Click the **Create Agent** button.
        3. Select **Start from a template**.
    </Step>
    <Step title="Browse and choose a template">
        You'll see the available templates organized by use case. Each one shows a brief description of what it does and the agent type (single prompt or conversational path).
    </Step>
    <Step title="Preview the template">
        Before using it, click on a template to see its full configuration:
        - **Prompt or conversational path**: the preconfigured conversation logic.
        - **Flows**: pre and post-call automations included.
        - **Integrations**: connections to external services used by the agent.
        - **Knowledge bases**: linked reference documents.

        This allows you to evaluate whether the template fits your use case before getting started.
    </Step>
    <Step title="Use the template">
        When you're ready, click **Use template**. The agent will load in the builder with all the template configuration applied.
    </Step>
</Steps>

## Customizing a template

Once the template is loaded, the agent is ready for you to adapt. The template highlights the fields that need your configuration to make the agent operational.

### Connections

The flows and integrations included in the template need your own credentials to work. Connect your accounts for the external services the agent uses (CRM, email, spreadsheets, etc.).

<Tip>
If you haven't set up connections yet, see the [Connections](/en/platform/connections) guide to learn how to authorize them.
</Tip>

### Knowledge bases

Add the relevant documents for your use case. Templates usually indicate what type of information the agent needs to respond correctly.

In **conversational path** agents, the template indicates which node is recommended for linking each knowledge base, so the information is available at the right moment in the conversation.

### Flows

If the template includes flows (pre or post-call automations), make sure they're connected to your services and adjust the steps according to your needs.

### Prompt and instructions

Adapt the prompt content to your business: company name, communication tone, specific information about your products or services, and any particular instructions.

## Publishing the agent

Once you've customized the template:

1. Click **Create Version** in the upper right corner.
2. Select **Publish when creating version**.
3. Select an existing number or [purchase a new one](/en/telephony/buy-numbers).
4. Click publish.

<Icon icon="party-popper" iconType="solid" /> Your template-based agent is ready to receive calls.
